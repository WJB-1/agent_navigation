# **Phase 5 工作总结报告**

## **📋 阶段概述**

**Phase 5** 是导航预览 Agent 验证 Demo 的最终阶段，实现**全链路联调与语音渲染**。本阶段将前端在 Phase 2 收集的起终点发送给后端，优雅地处理大模型漫长的思考时间，并将后端返回的文本转化为有节奏感、拟人化的语音播报，同时与地图进行视觉联动。

---

## **🎯 核心目标**

1. **请求调度与 Loading 状态**: 向后端发起推演请求，优雅处理 15-30 秒的超长等待时间
2. **TTS 语音引擎**: 利用浏览器原生 Speech API，将文本转化为有停顿、有节奏的专业播报
3. **推演播放器面板**: 展示推演结果文字与数据，提供 TTS 播报控制中心
4. **沙盘视觉联动**: 让地图视角跟着语音播报一起"走"，实现视听同步

### **模块 5.1 & 5.2: TTS 语音引擎 (TTSEngine)**

| 文件 | 说明 | 核心功能 |
|------|------|----------|
| `frontend/src/utils/ttsEngine.js` | TTS 语音合成引擎 | 支持智能断句、强制停顿、中文音色选择、队列播放控制 |

**核心技术特性:**

```javascript
// 智能断句 - 按换行符或句号拆分
_splitIntoSentences(text) {
    const paragraphs = text.split('\n').filter(p => p.trim());
    // ... 按标点符号分割
}

// 强制停顿 - 每句话之间停顿 0.8-1 秒（Phase 5 核心要求！）
utterance.onend = () => {
    const pauseDuration = 800 + Math.random() * 200; // 800-1000ms
    setTimeout(() => {
        this._playNext(options);
    }, pauseDuration);
};
```

**API 接口:**
```javascript
const ttsEngine = new TTSEngine();

// 播放
await ttsEngine.play(text, {
    rate: 0.9,        // 语速（稍慢，给视障者反应时间）
    pitch: 1,         // 音调
    volume: 1,        // 音量
    onSentenceStart: (index, sentence) => {},
    onSentenceEnd: (index, sentence) => {},
    onPlaybackEnd: () => {}
});

// 控制
ttsEngine.pause();
ttsEngine.resume();
ttsEngine.stop();
```

## **🚀 使用指南**

### **1. 启动服务**
```bash
# 1. 启动 Blind_map 后端 (端口 3001)
cd Blind_map/backend && npm start

# 2. 启动 navigation_agent 后端 (端口 3002)
cd navigation_agent/backend && npm start

# 3. 启动前端开发服务器 (端口 5173)
cd navigation_agent/frontend && npm run dev
```

### **2. 使用流程**
1. 在地图上右键选择"设为起点"
2. 在地图上右键选择"设为终点"
3. 点击"生成推演播报"按钮
4. 等待 15-30 秒（多智能体推演中）
5. 播放器面板弹出，点击"▶ 播放"开始语音播报
6. 地图上会显示黄色标记的关键节点

### **3. 模型选择**
- **推荐**: 阿里云百炼 `qwen-vl-plus`（稳定、支持多模态）
- **备选**: Google `gemini-2.0-flash`（快速、支持多模态）
- **仅文本**: DeepSeek `deepseek-chat`（纯文本、低成本）
